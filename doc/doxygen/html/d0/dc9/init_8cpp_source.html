73 //
74 // Thread management and startup/shutdown:
75 //
76 // The network-processing threads are all part of a thread group
77 // created by AppInit() or the Qt main() function.
78 //
79 // A clean exit happens when StartShutdown() or the SIGTERM
80 // signal handler sets fRequestShutdown, which triggers
81 // the DetectShutdownThread(), which interrupts the main thread group.
82 // DetectShutdownThread() then exits, which causes AppInit() to
83 // continue (it .joins the shutdown thread).
84 // Shutdown() is then
85 // called to clean up database connections, and stop other
86 // threads that should only be stopped after the main network-processing
87 // threads have exited.
88 //
89 // Note that if running -daemon the parent process returns from AppInit2
90 // before adding any threads to the threadGroup, so .join_all() returns
91 // immediately and the parent exits from main().
92 //
93 // Shutdown for Qt is very similar, only it uses a QTimer to detect
94 // fRequestShutdown getting set, and then does the normal Qt
95 // shutdown thing.
96 //
